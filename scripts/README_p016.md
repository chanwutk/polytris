# p016_tune_track_rate.py - Per-Tile Tracking Accuracy Evaluation

## Purpose

This script computes **per-tile tracking accuracy** at different frame sampling rates to determine optimal sampling thresholds for each tile position in the video dataset. The goal is to identify which tiles maintain good tracking performance at higher sampling rates (fewer frames processed) versus which tiles require lower sampling rates (more frames) for accurate object tracking.

## Overview

The script evaluates tracking accuracy by comparing a **ground truth tracker** (runs on all frames with `sample_rate=1`) against **sampled trackers** (run at sampling rates 1, 2, 4, 8, 16). For each tile position in the frame grid, it measures how well object associations are maintained when frames are sampled at different rates.

## Key Concepts

### Sampling Rates
- **sample_rate=1**: Process every frame (ground truth)
- **sample_rate=2**: Process every 2nd frame
- **sample_rate=4**: Process every 4th frame
- **sample_rate=8**: Process every 8th frame
- **sample_rate=16**: Process every 16th frame

### Association Correctness
An object detection is considered **correctly associated** if:
1. The tracker maintains the same object identity (track ID) across frames as the ground truth
2. When walking forward/backward in time, the detection chain remains consistent with ground truth

An object detection is **incorrectly associated** if:
- The tracker loses track of the object (association breaks)
- The tracker assigns a different object identity (ID switch)
- The sampled tracker misses the object entirely

## Algorithm

### Phase 1: Detection (uses p011 output)
1. Load all detections from `{CACHE_DIR}/{dataset}/indexing/segment/detection/{video}.detections.jsonl`
2. These detections were generated by p011_tune_detect.py on ALL frames
3. Scale detection coordinates to tile-aligned resolution

### Phase 2: Run Trackers
1. **Ground Truth Tracking** (sample_rate=1):
   - Run tracker on ALL frames
   - Get complete track associations for reference

2. **Sampled Tracking** (for each rate in [1, 2, 4, 8, 16]):
   - Run tracker with detections provided only at sampled frames
   - Non-sampled frames get empty detection arrays
   - Tracker still called on every frame to maintain Kalman filter state

### Phase 3: Evaluate Temporal Consistency

For each **sampled frame** F:

#### Step 3.1: Match Detections
- Use **Hungarian matching** with IoU threshold (default 0.3)
- Match GT detections to sampled detections on frame F
- Result: matched pairs `(gt_detection, sampled_detection)` with their track IDs

#### Step 3.2: Check Temporal Consistency

For each **matched pair** `(gt_track_id, sampled_track_id)` at frame F:

**Forward Walk:**
```
1. Start at frame F + sample_rate
2. While not past both tracks' frame boundaries:
   a. Check if gt_track has detection at current frame
   b. Check if sampled_track has detection at current frame

   c. If NEITHER has detection:
      → Continue walking (increment frame by sample_rate)

   d. If EXACTLY ONE has detection (not both):
      → Association BROKEN (incorrect)
      → Stop walking forward

   e. If BOTH have detections:
      → Check if they match each other on this frame (IoU matching)
      → If they match: Association MAINTAINED (correct), stop walking
      → If they don't match: Association BROKEN (incorrect), stop walking

3. If we reach end without finding any detection:
   → Both tracks ended naturally together (correct)
```

**Backward Walk:**
```
Same logic as forward walk, but starting at frame F - sample_rate
and walking backward in time.
```

**Result:**
- Detection is **CORRECT** only if BOTH forward AND backward checks pass
- Detection is **INCORRECT** if EITHER check fails

#### Step 3.3: Handle Unmatched GT Detections

For each **unmatched GT detection** at frame F:
- GT detected the object but sampled tracker missed it
- Count as **INCORRECT** (missed detection)

#### Step 3.4: Assign to Tiles

For each detection (matched or unmatched):
1. Determine all tiles the detection's bounding box overlaps
2. Increment correct/incorrect count for each overlapping tile

### Phase 4: Save Per-Video Counts

For each video and sample rate, save raw per-tile correct/incorrect
**counts** as partial results (shape `(num_rates, grid_h, grid_w, 2)`).

### Phase 5: Aggregate Across Videos

1. **Sum** per-video counts across all videos in the dataset
2. Compute Laplace-smoothed accuracy from the **aggregated counts**:
   ```
   accuracy = (total_correct + 1) / (total_correct + total_incorrect + 2)
   ```
   - Videos with more detections contribute proportionally more
   - Final shape: `(num_rates, grid_h, grid_w)`

## Input

- **Detections**: `{CACHE_DIR}/{dataset}/indexing/segment/detection/{video}.detections.jsonl`
  - Generated by p011_tune_detect.py
  - Format: `[frame_idx, [[x1,y1,x2,y2,score], ...], timing]` per line

## Output

For each `(dataset, tracker, tile_size)` combination:

```
{CACHE_DIR}/{dataset}/indexing/track_rate/{tracker}_{tile_size}/
├── accuracy.npy          # shape (5, grid_h, grid_w), float32
├── counts.npy            # shape (5, grid_h, grid_w, 2), int64 [correct, incorrect]
├── partial/              # per-video counts
│   ├── {video}.npy       # shape (5, grid_h, grid_w, 2), int64
│   └── ...
└── sample_rates.json     # [1, 2, 4, 8, 16]
```

**accuracy.npy**: Laplace-smoothed accuracy computed from aggregated counts, values in [0, 1]
**counts.npy**: Summed correct/incorrect counts across all videos
**partial/{video}.npy**: Per-video raw correct/incorrect counts per tile per sample rate

## Usage

```bash
# Inside Docker container
./run scripts/p016_tune_track_rate.py

# With custom settings
./run scripts/p016_tune_track_rate.py --iou_threshold 0.5 --num_workers 20
```

### Arguments

- `--iou_threshold`: IoU threshold for Hungarian matching (default: 0.3)
- `--num_workers`: Number of parallel workers (default: 40)

## Dependencies

### Scripts
- **p011_tune_detect.py** must be run first to generate detections on all frames

### Utilities (from polyis/utilities.py)
- `get_config()`: Loads global configuration
- `create_tracker()`: Creates SORT/ByteTrack tracker instances
- `register_tracked_detections()`: Registers tracker output to frame_tracks
- `get_video_resolution()`: Gets video dimensions
- `get_overlapping_tiles()`: Finds tiles overlapping a bounding box
- `ProgressBar`: Parallel processing with progress display

### External Libraries
- `scipy.optimize.linear_sum_assignment`: Hungarian matching algorithm
- `polyis.b3d.sort.iou_batch`: Vectorized IoU computation

## Example Output Interpretation

```python
import numpy as np
accuracy = np.load('accuracy.npy')  # shape (5, 18, 32)

# For tile at position (row=10, col=15), sample_rate=4 (index 2)
tile_accuracy = accuracy[2, 10, 15]
# If tile_accuracy = 0.95, this tile maintains 95% tracking accuracy at sample_rate=4
# If tile_accuracy = 0.60, this tile degrades significantly at sample_rate=4
```

**Interpretation:**
- **High accuracy** (>0.9) at a given sample rate → tile can use that rate safely
- **Low accuracy** (<0.7) at a given sample rate → tile needs lower rate (more frames)
- Use these per-tile thresholds to create a **spatially-adaptive sampling strategy**

## Notes

- Tracking is **CPU-only** (no GPU required for tracking phase)
- Parallelizes across videos (one worker per video)
- Memory efficient: processes one video at a time, saves partial results
- Total runtime depends on: number of videos, video length, number of trackers
